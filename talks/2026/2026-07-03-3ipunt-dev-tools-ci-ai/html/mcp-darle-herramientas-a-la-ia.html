MCP: darle herramientas a la IA

Los MCP que uso

MCP = conectar la IA a herramientas y datos externos.

  • context7 — documentación actualizada de librerías.
  • chrome-devtools / claude-in-chrome — navegador y DevTools.
  • github — issues, PRs y repos.
  • exa — búsqueda web.
  • filesystem — acceso a archivos.

Sin MCP la IA solo habla. Con MCP, actúa.

Notas del ponente

Diapositiva de poco texto: explícala tú, no la leas. Un MCP (Model Context Protocol) es un enchufe estándar entre el modelo y una herramienta.

Por cada uno, una frase:
- context7: le da los docs reales y recientes de la librería, así deja de inventarse APIs viejas o inexistentes. Es lo que más alucinaciones evita.
- chrome-devtools / claude-in-chrome: la IA abre el navegador, mira la consola, la red y el DOM → depurar web y automatizar pruebas.
- github: lee y escribe issues y PRs → convierte una conversación en trabajo trazable.
- exa: búsqueda web cuando necesita algo fuera del repo.
- filesystem: leer/escribir archivos fuera del proyecto actual.

Remata: eliges qué MCP montas por proyecto; no todos, solo los que ese trabajo necesita.
