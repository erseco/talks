Skills: recetas reutilizables

Capacidades empaquetadas

Una skill es un procedimiento o capacidad empaquetada que la IA invoca cuando toca. Las que uso:

  • superpowers — orquestación multi-agente: brainstorming, TDD, code review, debugging sistemático.
  • security-audit (Cloudflare) — auditoría de seguridad del código.
  • webwright — automatización y pruebas web con navegador real.
  • code-review, improve, modern-web-guidance — revisión, mejoras y buenas prácticas web al día.

Skills = procesos que la IA sigue igual cada vez; no dependes de recordar el prompt perfecto.

Notas del ponente

Diferencia clara: MCP = herramientas (con qué actúa); skill = receta/proceso (cómo trabaja). Se combinan.

Destaca los que quieres enseñar:
- security-audit de Cloudflare: pídele "haz una auditoría de seguridad de este código" y aplica una metodología, no improvisa.
- superpowers: convierte una tarea en un flujo con fases (planificar → implementar con TDD → revisar) y hasta lanza subagentes en paralelo.
- webwright: automatiza tareas web reproducibles con capturas de evidencia.

Si hay tiempo, lanza una skill en vivo (p. ej. una mini auditoría o un code-review del diff actual).
